Git workflow
1- Crear rama a partir de master
2- confirmaciones a esa rama
3- enviar esa rama a tu fork
4- abre un pull request en tu git hub
5- debatir
5.1- hacer commits
6- propietario cierra el pull request. (Fusionando si aceptó)
